### React is a Library

- Library for interactive UIs
- View-only
  - **Not** MVC
  - **Not** bundling
  - **Not** transpiling

---

### React is an **Architecture**

- React software outputs a dynamic tree
  - Of [DOM](https://en.wikipedia.org/wiki/Document_Object_Model) nodes
  - Of iOS controls
  - Etc...
- A **_renderer_** mediates between React and its in-memory tree
- **_Reconciliation_** makes the host tree match the in-memory tree

---

### React is an **Ecosystem**

- React is 'just a library'
- Most projects rely on a vast ecosystem for other things

---

### Things You Might Need

- **Dependency management** - [npm](https://www.npmjs.com/), [yarn](https://yarnpkg.com/)
- **Bundler** - Webpack, Rollup
- **Compiler** -- jsx, typescript, modern javascript
- **SSR** - Next.js
- **Development tools** - Hot reload, etc.
- **State management** - Zustand, Redux
- **Styling** - CSS / CSS-in-js
- **Routing** - React Router
- **HTTP** - React Query

## Creating a React App

---

### Option 0: Framework-Free

- All you really need--
  - **React** - For the core framework
  - **React-dom** - For web stuff
  - **Babel** - For JSX, and ES6+
- **Pros** No build process, Good for learning
- **Cons** No build process, no web server, not industrial strength

---

### Demo: Framework-Free

```html
<html>
  <head>
    <script src="https://unpkg.com/react@18/umd/react.development.js" crossorigin></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js" crossorigin></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
  </head>
  <body>
    <div id="root"/>
    <script type="text/babel">
      const App = () =>  <h1>Hello World!</h1>;
      ReactDOM.createRoot(document.getElementById('root'))
              .render(<App />);
    </script>
  </body>
</html>
```

---

### Vite: Pros and Cons

#### Pro

- Unopinionated (Relatively)
  - Good for learning plain React
  - Good for building pure old-school SPAs
- Great DX: Fast HMR, small bundles

#### Con

- Unopinionated (Relatively)
  - Still lots of configuration todo before production
- On your own for code splitting, routing, SSR

- Usually a project consists of
  - **Root** contains
    - Tooling configuration. e.g. `package.json`
    - **index.html**
  - **/public** contains everything _not_ managed by the build system
  - **/src** holds all executable code, styling, and assets

### File: `index.tsx`

- Entrypoint
- Loads the root component (often called **App**)

```ts
import React from 'react';
import { Root } from './App';

ReactDOM.render(
  <React.StrictMode>
    <Root />
  </React.StrictMode>,
  document.getElementById('root'),
);

```

---

### About [React.StrictMode](https://react.dev/reference/react/StrictMode)

- Turns on dev-mode
  - Double-rendering, double-effects
  - Checks for deprecations
